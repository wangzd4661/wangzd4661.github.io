ArrayDeque简介

在上篇LinkedList介绍了Deque双端队列接口。LinkedList是它的一个实现，而本篇文章的ArrayDeque是它另外的一个实现。可以通过名称来看，它是通过数组来实现的，不是类似LinkedList的链表结构。需要使用队列和栈的数据结构时，这是另外的选择，而且数组的读取性能还强于LinkedList的链表结构。

ArrayDeque源码分析

构造函数确定容量

先从构造函数来看，默认容量为16。如果指定了容量，则会通过calculateSize方法计算出一个大于指定容量的最小的2的幂次方的数。比如指定为10，则经过计算后，容量为16

以10为例子看calculateSize方法的实现

后面的操作已经和右移4位的结果一样。经过操作后10变为15。自增后为16。其实这段右移或操作的关心点是numElements二进制第一位为1的位。然后把这位后面的位都变为1。然后自增就得到这个数最小的2次幂。

添加元素

addLast方法很简单，直接在tail位置写入元素即可。
更新tail的位置

(tail + 1)很好理解，更新tail位置。& (elements.length - 1)的意义在于当tail更新到数组最后一个位置时改如何处理？elements.length - 1的二进制位全是1（为什么需要是2的幂）。当tail==elements.length时，&操作会让tail=0，回到起点。此时tail==head成立。进行扩容操作。

addFirst方法和addLast类似。但是head = (head - 1) & (elements.length - 1)注意当head=0时下标是从elements.length - 1开始的。如：-1&7=7

意味着head并非在前面，tail也并不是在后面。注意理解循环二字。& (elements.length - 1)操作是循环实现的主要体现。不太理解时。可以用debug调试观察变量的变化。

删除元素

删除元素的基本最终调用的都是pollFirst和pollLast

迭代器

ArrayDeque有两个迭代器，分别是DeqIterator和DescendingIterator，一个是正序，一个是倒序遍历。

在ArrayDeque的迭代器中没有使用类似ArrayList的modcount来在迭代过程中控制并发修改异常。使用了tail != fence || result == null简单的控制了并发修改异常。虽然不能控制所有情况，但是大部分常用情况都能控制到。

总结

  • ArrayDeque是一个用循环数组实现的双端队列。不支持null元素。
  • 可以当做栈和队列使用，因为基于数组，可以通过下标查找元素，而且不需要包装元素。效率高于LinkedList。
  • 没有提供删除中间元素的方法。
  • 扩容机制是变为成原来的数组的两倍大小。
  • 数组容量一定是2的幂次方（循环数组的关键），默认为16，最小为8。
